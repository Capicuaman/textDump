## Question 1: Skill Directory Discovery

You create a skill at `~/.claude/skills/my-tool/skill.md` and invoke it with `/my-tool`. It works perfectly. Then you create a project-specific version at `~/.claude/projects/-home-user-myproject/skills/my-tool/skill.md`. What happens when you invoke `/my-tool` from inside that project?

**A)** Error: duplicate skill names are not allowed
**B)** The global skill runs (first found wins)
**C)** The project-specific skill runs (project scope takes precedence)
**D)** You must specify which one: `/my-tool@global` or `/my-tool@project`

**‚úÖ CORRECT ANSWER: C**

**Explanation:**
Claude Code uses a **hierarchical precedence system** for skills:
1. **Project-specific skills** (highest priority)
2. **Plugin-provided skills**
3. **Global skills** (lowest priority)

When you're in a project and invoke `/my-tool`, Claude searches:
- First: `~/.claude/projects/[current-project]/skills/my-tool/`
- If not found: plugin skills
- If not found: `~/.claude/skills/my-tool/`

The project-specific version shadows the global one within that project context.

**Why Other Options Are Wrong:**
- **A:** Duplicate names are allowed - this is the entire point of skill scoping
- **B:** Global skills have the LOWEST priority, not the highest
- **D:** No such syntax exists - precedence is automatic based on context

**Key Takeaway:** Project-specific skills always override global skills within their project scope.

## Question 2: Project Key Generation

Which of these paths would generate the SAME project key?

```
/home/user/My Projects/app
/home/user/my-projects/app
/home/user/MY-PROJECTS/APP
```

**A)** All three generate the same key (case-insensitive, spaces normalized)
**B)** None of them - each has a unique key (case and spaces matter)
**C)** First and second are the same (space‚Üídash conversion) but third differs (case matters)
**D)** All three are invalid (spaces not allowed in project paths)

**‚úÖ CORRECT ANSWER: B**

**Explanation:**
Project keys are generated by:
1. Taking the absolute path
2. Converting to lowercase
3. Replacing `/` with `-`
4. Prefixing with `-`

After normalization:
- `/home/user/My Projects/app` ‚Üí `-home-user-my projects-app` (space preserved)
- `/home/user/my-projects/app` ‚Üí `-home-user-my-projects-app` (dash preserved)
- `/home/user/MY-PROJECTS/APP` ‚Üí `-home-user-my-projects-app` (lowercased, same as #2)

Paths 2 and 3 generate the same key after lowercasing, but path 1 differs due to the space.

**Why Other Options Are Wrong:**
- **A:** Spaces are NOT converted to dashes - only directory separators are
- **C:** Paths 2 and 3 are actually the same (both lowercase to same result)
- **D:** Spaces are valid in paths (though not recommended)

**Key Takeaway:** Project keys are case-insensitive (lowercased) but preserve other characters like spaces. Avoid spaces in project paths for consistency.

## Question 4: Skill Invocation Timing

You have a skill that reads project files and generates documentation. When you invoke `/document-project`, at what point does Claude Code load and read the skill.md file?

**A)** When Claude Code starts up (all skills preloaded)
**B)** When you type `/doc` (autocomplete triggers loading)
**C)** When you press Enter after typing `/document-project`
**D)** After you press Enter and Claude confirms the skill exists

**‚úÖ CORRECT ANSWER: C**

**Explanation:**
Skill loading is **lazy and on-demand**:
1. Skills are discovered and indexed when Claude Code starts (directory scan)
2. The actual skill.md content is NOT loaded until invocation
3. When you press Enter on `/skill-name`, Claude Code:
   - Reads the skill.md file
   - Injects its content into the prompt
   - Processes the skill's instructions

This lazy loading means you can have hundreds of skills without affecting startup time.

**Why Other Options Are Wrong:**
- **A:** Only skill names/paths are indexed at startup, not content
- **B:** Autocomplete uses the skill name index, doesn't load content
- **D:** Skill existence is checked before reading content, but the read happens at invocation time, not after confirmation

**Key Takeaway:** Skill content loads on-demand when invoked, not at startup. This keeps Claude Code fast even with many skills.

2. Claude Code reads: `skill.md` content

**Key Takeaway:** Skill arguments are automatically appended as `ARGUMENTS: [args]`. No template syntax needed in skill.md.
