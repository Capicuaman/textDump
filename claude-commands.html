Claude Code Commands

Master the CLI with Slash Commands

What Are Slash Commands?

Slash commands are special instructions that start with / in Claude Code (the CLI tool). They provide quick access to functionality like clearing conversations, managing tasks, configuring settings, and more.

Pro Tip: Type /help at any time to see all available commands in your CLI session.

Essential Commands

/help

Get Help and View Available Commands

Displays all available slash commands with brief descriptions.

Use Case: You're new to Claude Code and want to discover what commands are available.
Example:
/help - Shows complete command list
/help git - Shows help for git-related features
/clear

Clear Conversation History

Starts a fresh conversation, clearing all previous context. Useful when switching to a completely different task.

Use Case: You've finished debugging one feature and want to start fresh on a completely unrelated task without context pollution.
Warning: This removes all conversation history. Make sure you've saved any important information first.
/tasks

View Active Background Tasks

Lists all running background tasks including agents, shell processes, and async operations.

Use Case: You launched multiple agents in parallel and want to check their status or get their task IDs.
Example Workflow:
1. Start background task: "Run tests in background"
2. Check status: /tasks
3. View output: Use task ID from /tasks to retrieve results
/model

Switch AI Model

Change between different Claude models (Opus, Sonnet, Haiku) for the current session.

Use Case: Switch to Haiku for simple file searches to save costs, then switch back to Sonnet for complex refactoring.
Models:
/model sonnet - Balanced (recommended)
/model opus - Most capable
/model haiku - Fastest and cheapest
/quit or /exit

Exit Claude Code

Cleanly exits the CLI session.

Use Case: You're done with your coding session and want to close Claude Code properly.

Workflow Management Commands

/undo

Undo Last File Changes

Reverts the most recent file modifications made by Claude.

Use Case: Claude made changes to a file that broke your tests or weren't what you wanted.
Tip: Use version control (git) for more granular control over changes.
/reset

Reset Conversation and Context

Similar to /clear but may also reset additional state like selected files or project context.

Use Case: You want a completely fresh start, clearing both conversation history and any loaded context files.
/save

Save Current Conversation

Exports the conversation to a file for future reference or sharing.

Use Case: You had a productive debugging session and want to save the conversation as documentation.
Example:
/save debugging-session.md
Saves conversation as markdown file

Git & Version Control Commands

/commit

Create Git Commit

Triggers Claude to analyze changes and create a git commit with an appropriate message.

Use Case #1: You've made changes across multiple files and want Claude to create a well-formatted commit message following your repo's conventions.
Use Case #2: You want to commit your work but aren't sure how to describe the changes concisely.
What Claude Does:
1. Runs git status and git diff
2. Analyzes your recent commit messages for style
3. Creates an appropriate commit message
4. Adds files and commits with attribution
Tip: You can also say "create a commit" naturally in conversation instead of using the slash command.
/pr or /pull-request

Create Pull Request

Guides you through creating a pull request using the gh CLI tool.

Use Case: You've completed a feature branch and want to create a PR with a well-written summary and test plan.
What Claude Does:
1. Reviews all commits in your branch
2. Generates PR title and description
3. Creates test plan checklist
4. Pushes to remote and creates PR via gh CLI
/diff

Show Git Diff

Displays changes in your working directory or staged files.

Use Case: Quick way to review what's changed before committing.
/status

Show Git Status

Displays current git status including modified, staged, and untracked files.

Use Case: Check repository state before starting work or making commits.

Configuration Commands

/settings

View or Modify Settings

Opens settings interface to configure Claude Code behavior, API keys, model preferences, etc.

Use Case #1: Set default model to Haiku for cost savings on simple tasks.
Use Case #2: Configure hooks that run before/after certain actions.
Use Case #3: Set up MCP servers for extended functionality.
/config

Configure Project Settings

Set project-specific configurations like coding style, test commands, or build processes.

Use Case: Tell Claude to always run npm test before committing in this project.

Advanced Commands

/agents

Manage Specialized Agents

List, launch, or interact with specialized AI agents (e.g., code-reviewer, test-runner, explore).

Use Case #1: Launch an "explore" agent to thoroughly map out a large codebase.
Use Case #2: Run a "code-reviewer" agent to analyze your recent changes.
Available Agents:
- explore: Fast codebase exploration
- plan: Software architecture planning
- general-purpose: Complex multi-step tasks
/hooks

Manage Hooks

View or configure hooks (shell commands that execute on certain events like tool calls).

Use Case #1: Run linter automatically before any git commit.
Use Case #2: Trigger a notification when long-running tasks complete.
Hook Types:
- user-prompt-submit-hook
- pre-tool-call-hook
- post-tool-call-hook
/mcp

Model Context Protocol Servers

Manage MCP servers that extend Claude's capabilities (database access, APIs, custom tools).

Use Case: Connect Claude to your PostgreSQL database so it can query data directly.
Tip: MCP servers let Claude interact with external systems beyond the filesystem.
/skills

Custom Skills Management

Create, edit, or run custom skills (reusable command sequences or workflows).

Use Case #1: Create a "/deploy" skill that runs tests, builds, and deploys your app.
Use Case #2: Build a "/review-pr [number]" skill that fetches and analyzes a PR.

Context Management Commands

/add or /attach

Add Files to Context

Explicitly add files or directories to the conversation context.

Use Case: You're about to discuss refactoring and want Claude to have specific files loaded.
Example:
/add src/components/Button.tsx
/add src/styles/*.css
/remove or /detach

Remove Files from Context

Remove previously added files from conversation context to reduce token usage.

Use Case: You've finished working on a set of files and want to free up context for new files.
/context

View Current Context

Shows what files, directories, or context is currently loaded in the conversation.

Use Case: Verify what Claude currently has access to before asking a question.

Debugging & Inspection Commands

/debug

Enable Debug Mode

Shows additional information about Claude's reasoning, tool calls, and internal processes.

Use Case: Claude isn't behaving as expected and you want to see what's happening under the hood.
/tokens

View Token Usage

Displays how many tokens have been used in the current conversation.

Use Case: Monitor API costs or check if you're approaching context limits.
/log

View Session Logs

Shows detailed logs of the current session including all tool calls and responses.

Use Case: Debugging why a particular operation failed or reviewing what Claude did.

Practical Workflow Examples

Workflow: Feature Development

  1. Start Fresh: /clear to begin with clean context
  2. Add Context: /add src/features/auth/*
  3. Request: "Add OAuth login to the auth system"
  4. Review Changes: /diff
  5. Commit: /commit
  6. Create PR: /pr

Workflow: Bug Investigation

  1. Check Status: /status
  2. Enable Debug: /debug
  3. Explore Codebase: "Find where user authentication errors are handled"
  4. Review Findings: Ask Claude to explain the bug
  5. Fix: "Fix the authentication error handling"
  6. Test: "Run the test suite"
  7. Commit Fix: /commit

Workflow: Code Review

  1. View Changes: /diff
  2. Launch Agent: Use explore agent to understand impact
  3. Request Review: "Review these changes for security issues"
  4. Check Tasks: /tasks (if using background agents)
  5. Apply Suggestions: Discuss and implement fixes
  6. Undo if Needed: /undo if changes are wrong

Workflow: Learning New Codebase

  1. Launch Explore Agent: "Explore the codebase and explain the architecture"
  2. Check Progress: /tasks
  3. Ask Questions: "How does the routing system work?"
  4. Save Findings: /save codebase-notes.md
  5. Add Key Files: /add important files for reference

Pro Tips for Command Usage

1. Natural Language Works Too
You don't always need slash commands. Claude understands natural requests like "commit these changes" or "create a pull request."
2. Chain Commands in Conversation
You can request multiple actions: "Run the tests, and if they pass, create a commit and push to origin."
3. Use Agents for Complex Tasks
When exploring large codebases or planning complex features, explicitly request agents rather than doing it all in the main conversation.
4. Save Frequently
Use /save after productive sessions to build up a knowledge base of solutions and patterns.
5. Monitor Token Usage
Use /tokens periodically in long sessions to avoid hitting limits or excessive costs.
6. Customize with Hooks
Set up hooks for repetitive tasks like always running formatters before commits or notifying you when builds complete.
7. Use /clear Strategically
Don't clear too often - Claude's context understanding improves with conversation history. Clear when switching to completely different projects.

Common Patterns

Pattern: Rapid Prototyping
1. /clear - Fresh start
2. Describe feature naturally
3. /diff - Review changes
4. Iterate with feedback
5. /commit - Save progress
Pattern: Safe Experimentation
1. Create feature branch first (manually)
2. Let Claude make experimental changes
3. /diff - Review thoroughly
4. /undo if needed
5. /commit when satisfied
Pattern: Documentation Generation
1. /add src/**/*.ts - Load codebase
2. "Generate API documentation"
3. Review and refine
4. /save docs/api-reference.md
Pattern: Legacy Code Understanding
1. Launch explore agent for overview
2. /tasks - Monitor progress
3. Ask specific questions about components
4. /add key files for deep dive
5. /save learnings as documentation

Command Aliases & Shortcuts

Common Aliases

  • /q or /quit - Exit
  • /c or /clear - Clear conversation
  • /h or /help - Help
  • /s or /status - Git status
  • /d or /diff - Git diff

When NOT to Use Commands

Don't overuse /clear: You lose valuable context. Only clear when truly switching projects.
Don't rely solely on /undo: Use proper version control (git) for important work.
Don't skip /diff before /commit: Always review changes before committing.
Don't forget hooks can block commands: If operations fail unexpectedly, check your hooks configuration.

Building Your Own Skills

Skills are reusable custom commands you can create for your specific workflows. They're essentially saved sequences of operations or specialized prompts.

Example Skill: /review-security
1. Run security linter
2. Check for common vulnerabilities (SQL injection, XSS)
3. Review authentication and authorization
4. Generate security report
Example Skill: /deploy-staging
1. Run test suite
2. Build production bundle
3. Check for errors
4. Deploy to staging environment
5. Run smoke tests
Tip: Create skills for any workflow you repeat more than twice.
